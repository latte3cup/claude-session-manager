# wmux Landing Page Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a single-page dark-themed landing page for wmux, served via GitHub Pages from `docs/index.html`.

**Architecture:** Single self-contained HTML file with inline CSS and no external dependencies. Six sections: Hero, Features, Terminal Demo, Quick Start, FAQ, Footer. Responsive layout using CSS Grid and Flexbox.

**Tech Stack:** HTML, CSS, vanilla JS (minimal — only for smooth scroll if needed). No frameworks, no build tools.

**Spec:** `docs/superpowers/specs/2026-03-23-website-design.md`

### Task 1: HTML skeleton with meta tags and base styles

**Files:**
- Create: `docs/index.html`

- [ ] **Step 1: Create the HTML file with document structure**

Create `docs/index.html` with:
- `<!DOCTYPE html>`, `<html lang="en">`, `<head>`, `<body>`
- Meta tags: charset, viewport, description, color-scheme, Open Graph (`og:title`, `og:description`, `og:type`)
- `<title>wmux — tmux for Windows</title>`
- Inline `<style>` block with CSS reset and base styles:
  - `*, *::before, *::after { box-sizing: border-box; margin: 0; padding: 0; }`
  - `html { scroll-behavior: smooth; }` with `@media (prefers-reduced-motion: reduce) { html { scroll-behavior: auto; } * { transition: none !important; } }`
  - `body { background: #0a0a0a; color: #fafafa; font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif; line-height: 1.5; }`
  - CSS custom properties: `--bg: #0a0a0a; --fg: #fafafa; --muted: #888; --accent: #3B82F6; --border: #222; --card-border: #333; --mono: 'Cascadia Code', 'Fira Code', 'SF Mono', 'Consolas', monospace;`
  - Container: `.container { max-width: 960px; margin: 0 auto; padding: 0 1.5rem; }`
  - Section spacing: `section { padding: 5rem 0; }` (override for hero: `#hero { padding: 0; }` since `.hero` has its own padding)
  - Section title style: `.section-title { font-size: 0.75rem; text-transform: uppercase; letter-spacing: 0.1em; color: var(--muted); margin-bottom: 2rem; }`
- Empty section landmarks: `<section id="hero">`, `<section id="features">`, `<section id="demo">`, `<section id="quickstart">`, `<section id="faq">`, `<footer>`

```html
  <div class="faq-list">
    <details class="faq-item">
      <summary>What shells does wmux support?</summary>
      <p>PowerShell, cmd, WSL bash, nushell, or anything. Set with the <code>--shell</code> flag or <code>WMUX_SHELL</code> environment variable.</p>
    </details>
    <details class="faq-item">
      <summary>What Windows versions work?</summary>
      <p>Windows 10 1809+ (ConPTY required). Best experience in Windows Terminal.</p>
    </details>
    <details class="faq-item">
      <summary>How is this different from tmux in WSL?</summary>
      <p>wmux is native Windows. No Linux subsystem, no translation layer. Your Windows shells, your Windows paths, your Windows tools.</p>
    </details>
    <details class="faq-item">
      <summary>Can AI agents use wmux?</summary>
      <p>Yes. The JSON-RPC socket API lets agents spawn terminals, send commands, and read output programmatically via named pipes.</p>
    </details>
    <details class="faq-item">
      <summary>Is it free?</summary>
      <p>Yes, MIT licensed. Open source forever.</p>
    </details>
  </div>
</div>
```
